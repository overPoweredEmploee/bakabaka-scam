# HTML CSS

# 1. Верстка макета

**ТЗ**:

Цель:
Сверстать одностраничный лендинг по макету Wishbone в Figma

Материалы:

- Макет: Wishbone (Figma). [Ссылка на макет](https://www.figma.com/design/ODvu3zvuKToCLy4mwq5JfI/Wishbone?node-id=0-1&t=UlrN4biMI55IEUPj-1)
- Репозиторий: ветка main, работа в отдельной ветке feature/wishbone

Требования к HTML:

- Семантическая структура: header, nav (если есть меню), main, section, footer; по необходимости — h1–h3, p, ul/ol, img.
- Минимизировать лишние обертки: без «дивов ради дивов».
- Корректные атрибуты: alt для изображений.
- Валидность: без критических ошибок валидатора.

Требования к CSS:

- Макетирование: Flexbox для сеток; position — только где нужно (бейджи/иконки/оверлеи).
- Методология: БЭМ (block\_\_element--modifier), понятные имена классов.
- Сброс базовых стилей стилей
- Шрифты:
  Вариант А: подключить через Google Fonts.
  Вариант Б\*: локально через @font-face, добавить системные fallback-шрифты.
- Переменные: завести CSS‑переменные для основных цветов и отступов.

Адаптив\*:

- Мобильный‑first.

Доступность и UX:

- Изображения с осмысленным alt.

Код‑стайл:

- Настроить и применить Prettier.
- Соблюдать DRY: переиспользовать переменные, общие утилити‑классы, не дублировать правила.

Процесс сдачи:

- Создать ветку feature/wishbone, коммиты небольшие и понятные (например: feat: header, layout). Почитать [Соглашение о коммитах](https://www.conventionalcommits.org/ru/v1.0.0/).
- Открыть Pull Request в main с чек‑листом. Ничо не тыкай, сам натыкаю:
  [ ] Семантическая структура
  [ ] БЭМ‑классы
  [ ] Reset/normalize
  [ ] Подключение шрифтов (GF или @font-face)
  [ ] CSS‑переменные для цветов/отступов
  [ ] Фокус‑стили, alt для изображений
  [ ] Prettier применен
